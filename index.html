La cursada

Como sucede cada semestre, los alumnos que se inscribirse en la Escuela DaVinci, inician un camino de conocimiento y desafíos al iniciar la carrera de Diseño y Programación Web. A lo largo de los tres años de cursada intensiva, aprenderán a desarrollar aplicaciones para la Web y dispositivos móviles, ambas muy demandadas en el mercado actual.

MAQUETADO Y DESARROLLO WEB

La materia se cursa dos veces por semana, con una duración de 2 hs. cada módulo; A lo largo de las 16 semanas que tiene el semestre –entre feriados y días destinados a prácticas y parciales– el alumno habrá tenido aprox. 27 clases que abarcarán distintos contenidos fundamentales para todo desarrollo a futuro y troncales para los cuatrimestres siguientes. El plan de estudios de la materia abarca los principales ítems de un sitio web funcional.

  • Lenguaje HTML, reglas sintácticas e inserción de código.
  • Estructura básica de un documento, elementos de texto, navegación entre distintas páginas.
  • Inserción de imágenes
  • Introducción al lenguaje CSS, implementación de estilos y diseño.
  • Problemas comunes en la estandarización entre navegadores.
  • Inserción y optimización de elementos multimediales (sonido, video, mapas de Google).
  • Generación y uso de formularios.
  • CSS Avanzado.
  • Publicación Online del sitio.

Todo esto, bajo los estándares del XHTML 1.0 que se encuentra regulado por la w3c mediante el uso de una herramienta propia del organismo llamada HTML Validator.

FUTURO LABORAL

De ellos, algunos saldrán expertos maquetadores, otros serán eximios diseñadores y algunos tendrán las nociones requeridas para desarrollar sistemas de mayor complejidad. Como dijo la prof. Mabel García en la primera clase El objetivo de la carrera es entrenar profesionales. No obstante, los roles de diseñador, maquetador y desarrollador web no siempre se delimitan con claridad. La confusión surge debido a que, durante el desarrollo de un proyecto web, es normal que algunos miembros del equipo de trabajo desempeñen más de un rol al mismo tiempo. Sin embargo, las atribuciones particulares de cada puesto pueden entenderse más fácilmente si conocemos la diferencia entre front-end y back-end.
